【CAS 认证流程 - Vue 前端】
参与角色：User (用户), Vue (前端), SpringBoot (后端), CAS (认证中心), Redis (缓存), JWT (令牌服务)。
认证步骤：
1. 用户访问 Vue 前端页面。
2. Vue 请求 SpringBoot 后端数据（无 Token）。
3. SpringBoot 返回重定向指令，引导跳转至 CAS 登录页。
4. CAS 展示登录表单，用户提交用户名和密码。
5. CAS 验证凭证，将登录状态（TGT）存入 Redis。
6. CAS 请求 JWT 服务生成 Token。
7. CAS 重定向回 Vue 前端，并携带 JWT Token。
8. Vue 携带 Token 再次访问 SpringBoot。
9. SpringBoot 请求 JWT 服务验证 Token 有效性，并查询 Redis 验证 TGT 状态。
10. 验证通过，SpringBoot 返回业务数据。

【CAS 认证流程 - UniAPP 移动端】
参与角色：User (用户), UniAPP (前端), SpringBoot (后端), CAS (认证中心), Redis (缓存), JWT (令牌服务)。
认证步骤：
1. 用户打开 UniAPP，App 检查本地登录状态。
2. 无 Token，App 打开内置浏览器跳转 CAS 认证页。
3. 用户在 CAS 页面输入凭证登录。
4. CAS 验证通过，存入 TGT 到 Redis，生成 JWT Token。
5. CAS 通过 URL Scheme 回调 UniAPP，传递 Token。
6. UniAPP 本地存储 Token。
7. 用户访问受保护内容，UniAPP 请求携带 Token。
8. SpringBoot 验证 JWT 和 Redis TGT 状态，返回数据。

【CAS 认证流程 - 微信小程序】
参与角色：User (用户), WeChatMP (小程序), SpringBoot (后端), CAS (认证中心), Redis (缓存), JWT (令牌服务)。
认证步骤：
1. 用户进入小程序，小程序调用 wx.login 获取 code。
2. SpringBoot 检查 code 绑定状态，未绑定则返回未认证。
3. 小程序通过 web-view 跳转 CAS 网页授权。
4. 用户在 CAS 页面登录，CAS 存入 TGT，生成 JWT 和静默 code。
5. CAS 通过 postMessage 将 Token 回传给小程序。
6. 小程序提交 code 和 token 给 SpringBoot 进行绑定。
7. SpringBoot 验证 Token 和 TGT，绑定成功返回 session_key。
8. 小程序存储凭证，后续请求携带 Token，后端进行复合验证。
